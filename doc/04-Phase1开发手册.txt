2.2 package.json（草案）

  {
    "name": "com.figma-unity.ui",

    "com.figma-unity.ui": "file:com.figma-unity.ui"

2.3 asmdef 要点

  Runtime：仅 UnityEngine，无 Editor 引用
  Editor：引用 Runtime + UnityEditor + TMP
  Tests：引用 Editor，optional UnityEngine.TestRunner

三、端到端流水线
----------------

  [用户] 选择导出包目录 exportDir
       │
       ▼
  FigmaExportPackage.Locate(exportDir)
       │  找到 *-full.json + 同目录 png
       ▼
  FigmaToIRConverter.Convert(exportDir, importSettings)
       │  解析 metadata + node
       │  AssetResolver 复制 png → Assets/UI/Generated/{screen}/
       │  输出 IRNode 根树（内存）
       ▼
  IRValidator.Validate(root) → warnings[]
       │
       ▼
  PrefabBuilder.Build(root, settings)
       │  递归：RectTransform → 视觉 → IRBinding → children
       ▼
  PrefabUtility.SaveAsPrefabAsset → Assets/UI/Generated/{screen}.prefab
       │
       ▼
  ImportReport.Log（节点数、降级、缺失资源）

  v1 策略：每次导入**全量覆盖** Prefab，不做增量 diff。

四、IR 数据模型（C# 草案）
--------------------------

  namespace FigmaUnity.UI.IR
  {
      class IRNode
      {
          string version;
          string id;
          string type;              // frame | text | image
          float x, y;
          float width, height;
          string anchor;            // top-left
          List<IRFill> fills;
          IRStroke stroke;
          float opacity;
          float cornerRadius;
          IRLayout layout;
          IRText text;
          Dictionary<string, object> meta;
          List<IRNode> children;
      }

      class IRFill   { string type; string color; float opacity; }
      class IRStroke { string color; float width; float opacity; }
      class IRText   { string content; float fontSize; string fontFamily;
                       string color; string align; bool bold; bool italic; }
      class IRLayout { string type; float gap;
                       float paddingTop, paddingRight, paddingBottom, paddingLeft;
                       string align; Dictionary<string, object> meta; }
  }

  Figma v2 专用模型（仅 Editor 转换层使用）：
    FigmaExportDocument { FigmaMetadata metadata; FigmaNode node; }
    FigmaNode 字段与 full.json 一一对应（irId, id, type, x, y, fills, layout...）

九、降级矩阵（v1 行为）
------------------------

  类型/场景              Unity 行为                         报告
  ─────────────────────  ─────────────────────────────────  ──────
  VECTOR                 RawImage 透明占位                  Warning
  LINE                   1~2px Image，描边色                Info
  DROP_SHADOW            不创建 Shadow                      Warning 汇总
  cornerRadius > 0       无圆角 Sprite 则方形 Image         Warning
  fontFamily 无映射      TMP 默认字体                       Warning
  visible = false        GameObject.SetActive(false)        Info
  layout space-between   H/V LayoutGroup + 子项宽设置       需 LayoutBuilder 特判

十、Canvas / Prefab 约定
------------------------

  生成 Prefab 根节点：
    - RectTransform：1080×2340，pivot (0.5,0.5)，anchoredPosition (0,0)
    - 可选挂 Canvas + CanvasScaler（仅当导入到场景时由 Importer 处理）

  CanvasScaler（若挂 Canvas）：
    UI Scale Mode = Scale With Screen Size
    Reference Resolution = 1080 × 2340
    Match = 0.5

  导入模式：
    A) 仅生成 Prefab 资产（默认）
    B) 同时实例化到当前场景指定 Canvas 下（Importer 可选项）

十三、常见问题
--------------

  Q: 导出包在仓库 figmajson/ 下，Unity 工程在 UITest/，如何引用？
  A: Importer 用绝对或相对路径 Browse；或复制 export 包到 UITest/Assets/StreamingAssets/FigmaImports/ 仅供测试。

  Q: 必须先写 .ir.json 文件吗？
  A: 否。v1 推荐内存 IRNode 直传 PrefabBuilder；可选 Debug 菜单落盘 .ir.json 便于 diff。

  Q: LayoutGroup 与绝对 x/y 是否冲突？
  A: 有 layout 的节点仍挂 LayoutGroup；子节点 x/y 由 Figma 导出，v1 以 RectTransform 绝对位置为准，
     LayoutGroup 主要影响运行时动态内容，静态复刻优先保证 anchoredPosition 正确。

  Q: irId 含 __6_5 后缀是否太长？
  A: 保持与 Figma 导出一致，不做二次改名；Phase 2 回导依赖 figmaNodeId 为主键。
