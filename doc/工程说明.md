# Figma ↔ Unity UI 互转探索工程 — 说明文档

> 更新：2026-07-01  
> **定位**：本仓库为 Figma 与 Unity UI 双向同步的**技术探索与原型验证**工程，用于验证格式、流程与字段覆盖；**非正式交付工具**。正式产品化需另行程序开发与 Figma 插件交付。

---

## 一、项目目标

在 Figma 设计稿与 Unity UI Prefab 之间建立可往返的 UI 复刻链路：

| 方向 | 说明 | 状态 |
|------|------|------|
| **Figma → Unity** | 设计稿导出 → Unity 生成 Prefab | Phase 1 已验收 |
| **Unity → Figma** | Prefab 改动导出 → Figma 按 id 回写 | 原型已打通，持续迭代 |

中间表示层为 **IR**（内存 `IRNode` 树），对外交换格式为 **`*-full.xml` / `*-full.json`**（Figma 全量导出）与 **`*-unity-export.xml`**（Unity 增量合并导出）。

---

## 二、整体工作流

```
┌─────────────────────────────────────────────────────────────────┐
│  Figma 设计稿                                                    │
└────────────┬────────────────────────────────────────────────────┘
             │  Tool 1: Full Design Exporter
             ▼
      *-full.xml / *-full.json + PNG 资源
             │
             ▼
┌────────────────────────────────────────────────────────────────┐
│  Unity Editor                                                   │
│  Tools > Figma UI Importer  →  IRNode  →  PrefabBuilder  → Prefab │
│  （程序编辑：布局、文案、锚点、Button 等）                          │
│  Tools > Figma UI Exporter  →  以 full 为模板 Patch 合并         │
└────────────┬───────────────────────────────────────────────────┘
             │  *-unity-export.xml（+ 可选 PNG）
             ▼
      Tool 2: Unity XML/JSON Importer
             │
             ▼
┌─────────────────────────────────────────────────────────────────┐
│  Figma 设计稿（按 id 更新）                                       │
└─────────────────────────────────────────────────────────────────┘
```

**身份匹配**：Figma 节点 `id` ↔ Unity `IRBinding.figmaNodeId`；`irId` 为历史/调试字段。

**重要架构差异**：

- Figma 侧为**全量导出**（完整节点树 + 全部 Figma 字段）。
- Unity Import 为**选择性映射**到 IR / Prefab（可通过 Import Profile 勾选）。
- Unity Export **不是全量**：以 `*-full.xml` 为模板做 Patch 合并；未写入的 Figma 字段靠模板原样保留。
- Unity 独有内容（Button、自定义脚本）靠 Prefab **Merge** 模式保留，不会进入 Figma XML。

## 四、Figma 侧工具（外部插件，规范在本仓库 doc/ 中）

本仓库**不包含** Figma 插件源码，仅维护对接规范与样例数据。插件分两个：

### Tool 1 — Full Design Exporter

- 选中 Frame/Component/Group，导出 `*-full.xml`（推荐）或 `*-full.json`。
- 全量属性：几何、视觉、layout、constraints、TEXT、组件属性等。
- 主键：`id`（如 `"6:124"`）。
- 图片：`IMAGE` fill 中 `imageFile`/`imageHash` 可为 null，由 Unity 用同目录 PNG + 文件名管理。

### Tool 2 — Unity XML/JSON Importer

- 上传 Unity 返回的 `*-unity-export.xml`（或 JSON）。
- 按 `id` 匹配，勾选同步项后更新 Figma 画布。
- 支持：属性更新、re-parent、新增/删除节点。

详见 `doc/figma工具说明.TXT`、`doc/07-v3-Figma工具对接.txt`。

## 五、Unity Package：`com.figma-unity.ui`

**位置**：`UITest/Packages/com.figma-unity.ui/`  
**Unity 版本**：2022.3 LTS  
**依赖**：UGUI、TextMeshPro、Newtonsoft.Json

### 5.1 Editor 菜单

| 菜单 | 类 | 作用 |
|------|-----|------|
| `Tools > Figma UI Importer` | `IRImporterWindow` | 读取 full XML/JSON，生成或合并 Prefab |
| `Tools > Figma UI Exporter` | `IRExporterWindow` | 读取 Prefab，Patch 合并写出 unity-export |

### 5.2 模块划分

| 目录 | 职责 |
|------|------|
| `Editor/Figma/` | Figma XML/JSON 反序列化、`FigmaToIRConverter`、资源解析 |
| `Editor/IR/` | `IRNode` 数据模型、校验、读取 |
| `Editor/Build/` | `PrefabBuilder` / `PrefabMerger`、坐标/约束/布局/文字/图片构建 |
| `Editor/Export/` | Prefab → Figma 文档 Patch（坐标逆变换、scale、约束、文案、图片、Auto Layout） |
| `Editor/Config/` | `ImportProfile` / `ExportProfile` 预设与选项 |
| `Editor/UI/` | Importer/Exporter 窗口、字体安装 |
| `Runtime/` | `IRBinding` 组件（挂于 Prefab 节点，记录 `figmaNodeId` / `irId`） |

### 5.3 Import Profile 预设

- **Visual Merge** — 日常推荐：Merge + 保留锚点 + 同步布局/视觉
- **Full Import** — 约束 + FILL 布局 + 字体 + 对齐
- **Static Absolute** — 纯左上角绝对坐标，首次导入

**Rebuild Mode**：`Merge`（保留 Button/脚本）或 `Replace`（全量重建，会丢失自定义组件）。

### 5.4 Export Profile 预设

- **Default** — 位置/可见/文案/对齐 + layoutAdjustments
- **Layout + Text** — 明确不同步 fills/typography

可选写回：constraints、typography、fills、image assets、删除节点等。

## 九、已验证能力摘要

- Figma full XML/JSON → Unity Prefab 全量导入（Frame / Text / Image / Auto Layout 部分支持）
- Prefab Merge 模式：保留 Button、自定义脚本，仅更新 IR 绑定节点
- Unity → Figma Patch 导出：位置/尺寸/旋转/scale、可见性、文案、对齐、constraints（可选）
- 图片资源按 `imageFile` 文件名同步
- 复杂视觉（描边/渐变/阴影）降级为 `_bakedPaint` 烘焙图
- 字体映射（Font Mapping Asset）与 TMP 对齐

---

## 十、明确不在范围内

- Lua / xLua / UI 事件绑定
- 增量 diff 更新（v1 为全量重建或 Merge）
- ScrollRect / Mask 100% 一致
- Figma 组件变体、混合文字 segments 完整双向
- 生产级错误处理、批处理流水线、CI 集成

---

## 十一、推荐使用流程

1. Figma 导出 `*-full.xml` + 同目录 PNG
2. Unity 首次 Import：`Static Absolute` + `Replace`
3. 在 Prefab 上挂交互组件、调整锚点与适配
4. Unity Export（`Default` Profile）→ 得到 `*-unity-export.xml`
5. Figma Tool 2 导入回写
6. 策划/UI 改 Figma 后，Unity 再 Import（`Visual Merge`）同步视觉与布局

**注意**：Unity 自建节点 Export 前应将 Anchors/Pivot 设为左上角 `(0, 1)`，否则 Figma 侧位置会偏移（见 `doc/06`、`doc/11`）。

| 本仓库（探索） | 正式工具（待交付） |
